# 服务器配置

```toml
[server]
host = "127.0.0.1"
port = 3000
workers = 0
```

## 什么时候需要改

- 本机测试：一般保持默认即可
- Docker 或容器：通常把 `host` 改成 `0.0.0.0`
- 反向代理部署：端口可以继续用 `3000`
- 机器负载较高时：再考虑调整 `workers`

## 字段说明

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `host` | string | `"127.0.0.1"` | 绑定地址；容器部署通常设为 `0.0.0.0` |
| `port` | u16 | `3000` | HTTP 监听端口 |
| `workers` | usize | `0` | 工作线程数；`0` 表示自动选择 |

## 常见写法

### 本机测试

```toml
[server]
host = "127.0.0.1"
port = 3000
workers = 0
```

### Docker 或容器

```toml
[server]
host = "0.0.0.0"
port = 3000
workers = 0
```

## 部署建议

- 不确定时，先保持 `workers = 0`
- 对外访问是否稳定，通常更依赖反向代理和 HTTPS 配置
- 如果你已经用了 Caddy 或 Nginx，应用本身继续监听本地端口即可
