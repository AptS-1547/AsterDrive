# 常用流程

## 新部署后的第一次管理员检查

第一次登录后，建议按这个顺序检查:

1. `管理 -> 概览`
2. `管理 -> 用户`
3. `管理 -> 存储策略`
4. `管理 -> 系统设置`
5. 左侧 `WebDAV`

重点确认:

- 默认存储策略是否正确
- 新用户默认配额是否合理
- 回收站保留天数和历史版本数量是否符合预期
- WebDAV 是否需要开启
- 如果要用 Gravatar，头像地址是否可访问

## 上传并整理文件

日常整理资料时，通常这样做:

1. 先建几个基础文件夹
2. 上传文件或文件夹
3. 用拖拽把它们移动到目标目录
4. 用多选完成批量移动、复制或删除

## 发送资料给别人

发资料时，通常这样做:

1. 在文件或文件夹上创建分享链接
2. 需要保密时设置密码
3. 需要时效控制时设置过期时间
4. 需要限制传播时设置下载次数
5. 去 `我的分享` 里复制链接并检查公开页是否正常

如果只是想修改密码、过期时间或下载次数，直接在 `我的分享` 里编辑原有链接即可。

## 继续未完成的上传

如果大文件上传中断，常见处理方式是:

1. 回到原来的文件夹
2. 点击继续上传
3. 重新选择同一个文件

只要上传会话还没过期，系统会尽量继续未完成的部分，而不是从头开始。

## 给设备创建 WebDAV 账号

如果你要在 Finder、Windows 资源管理器、rclone 或同步工具里接入 AsterDrive，推荐这样做:

1. 在 `WebDAV` 页面创建一个专用账号
2. 按需要限制到某个目录
3. 把 WebDAV 地址、用户名和密码填进客户端
4. 先做一次真实读写测试

这样某台设备停用时，只需要停用这一条 WebDAV 账号，不影响网页登录密码。

## 处理误删

普通删除后的项目会先进回收站。最常见的处理方式是:

1. 误删后直接去回收站恢复
2. 不再需要的项目在回收站里永久删除
3. 管理员定期确认回收站保留天数是否合理

## 给新用户分配空间和策略

新增用户后，通常这样处理:

1. 在 `管理 -> 用户` 里创建账号，或找到现有账号
2. 设置这个用户的总配额
3. 给这个用户分配可用的存储策略
4. 在多个策略里指定默认策略

这样可以做到:

- 大部分用户走默认本地存储
- 某些用户单独走 S3
- 某些账号有更大的容量额度
