# 管理面板

这一页描述当前 `frontend-panel/` 已经落地的页面和它们对应的后端能力。

## 路由总览

| 路由 | 权限 | 作用 |
| --- | --- | --- |
| `/login` | 公开 | 登录页 |
| `/` | 登录后 | 文件浏览器 |
| `/trash` | 登录后 | 回收站 |
| `/settings/webdav` | 登录后 | WebDAV 账号管理 |
| `/s/:token` | 公开 | 公开分享页 |
| `/admin/users` | 管理员 | 用户与用户策略管理 |
| `/admin/policies` | 管理员 | 存储策略管理 |
| `/admin/shares` | 管理员 | 全站分享审计 |
| `/admin/locks` | 管理员 | WebDAV / 资源锁管理 |
| `/admin/settings` | 管理员 | 运行时配置管理 |

## 文件浏览器

文件浏览器页由这些能力组成：

- 左侧文件夹树
- 文件 / 文件夹列表
- 上传拖拽区
- 文件预览
- 分享弹窗
- 版本历史弹窗

当前前端已经接入三种上传模式：

- `direct`
- `chunked`
- `presigned`

其中分片上传支持：

- 本地 `localStorage` 保存 session
- 查询服务端 `chunks_on_disk`
- 失败后继续续传

## 回收站页

`/trash` 目前支持：

- 文件与文件夹分别展示
- 恢复单个条目
- 永久删除单个条目
- 一键清空回收站

对应接口是整套 `/api/v1/trash/*`。

## WebDAV 账号页

`/settings/webdav` 对应当前 WebDAV 账号管理接口：

- 列表
- 创建
- 启用 / 停用
- 删除
- 测试凭据

创建时可以选择：

- 自定义密码
- 留空让后端随机生成一次性密码
- 限制到某个根文件夹

## 管理后台

### 用户页

`/admin/users` 当前支持：

- 浏览所有用户
- 修改角色、状态、总配额
- 为用户分配多个存储策略
- 设置某个用户策略为默认
- 调整每个策略项的额度

### 策略页

`/admin/policies` 当前支持：

- 创建本地或 S3 策略
- 编辑已存在策略
- 测试策略连通性
- 设置默认策略
- 删除策略

注意当前后端实现里：

- 创建策略时 `chunk_size` 仍会先写成固定的 `5 MiB`
- 若要精确调整，需创建后再编辑
- `driver_type` 创建后不能通过现有 PATCH 接口修改

### 分享页

`/admin/shares` 用于：

- 浏览全站分享
- 快速打开公开链接
- 直接删除任意分享

### 锁页

`/admin/locks` 用于：

- 查看当前全部资源锁
- 强制解锁单个资源
- 清理全部过期锁

### 设置页

`/admin/settings` 是 `system_config` 的前端入口，当前对后端真正生效的键包括：

- `default_storage_quota`
- `webdav_enabled`
- `trash_retention_days`
- `max_versions_per_file`

## 公开分享页

`/s/:token` 是一个独立的公开页面，不依赖登录态。

当前表现：

- 文件分享：展示下载按钮、下载次数、过期时间
- 文件夹分享：只展示分享根目录的内容
- 密码分享：先验证密码，再写入短期 HttpOnly Cookie

## 交付方式

这套前端最终会被打包进后端二进制，但运行时仍会优先读取当前工作目录下的 `./frontend-panel/dist`。部署时如果你发现页面与预期版本不一致，先检查工作目录里是否存在这套目录。
