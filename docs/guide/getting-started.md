# 快速开始

下面这套流程按 Docker 编写，适合第一次把 AsterDrive 跑起来做验证。

## 1. 启动服务

```bash
docker run -d \
  --name asterdrive \
  -p 3000:3000 \
  -e ASTER__SERVER__HOST=0.0.0.0 \
  -e ASTER__AUTH__COOKIE_SECURE=false \
  -e ASTER__DATABASE__URL="sqlite:///data/asterdrive.db?mode=rwc" \
  -v asterdrive-data:/data \
  ghcr.io/apts-1547/asterdrive:latest
```

第一次启动时，AsterDrive 会自动完成这些准备工作：

- 在当前工作目录生成 `config.toml`
- 创建默认 SQLite 数据库 `asterdrive.db`
- 创建默认本地上传目录 `data/uploads`
- 自动执行数据库迁移
- 自动创建默认本地存储策略 `Local Default`
- 初始化管理后台里会用到的系统设置

默认地址：

```text
http://服务器地址:3000
```

上面的命令里把 `auth.cookie_secure` 临时设成了 `false`，这样本机或内网用 HTTP 测试时可以正常登录。正式上线到 HTTPS 域名后，把它改回 `true`。

## 2. 打开页面并创建第一个账号

1. 在浏览器打开 `http://服务器地址:3000`
2. 输入用户名、邮箱和密码
3. 完成首次创建后登录

第一个创建出来的账号会自动成为管理员。

## 3. 上传并确认第一个文件

登录后先做三件事：

- 创建一个测试文件夹
- 上传一个文件
- 打开它，确认可以预览或下载

## 4. 试一次分享

在文件或文件夹的操作菜单里创建分享链接。可以按需设置：

- 密码
- 过期时间
- 最大下载次数

把链接粘到无痕窗口或另一台设备里，确认公开访问页面能正常打开、预览或下载。

## 5. 如果要用 WebDAV，再做一次连接测试

如果你要在 Finder、Windows 资源管理器、rclone 或其他客户端里使用 AsterDrive：

1. 打开 `WebDAV`
2. 创建一个专用 WebDAV 账号
3. 复制地址、用户名和密码
4. 在客户端里做一次真实连接和读写测试

密码只会在创建成功时显示一次，建议当场保存。

## 6. 管理员首次检查

建议第一次登录后把下面几项检查一遍：

- `管理 -> 用户`
- `管理 -> 存储策略`
- `管理 -> 系统设置`
- `WebDAV`

建议至少确认：

- 默认存储策略是否正确
- 新用户默认配额是否合适
- 回收站保留天数和历史版本数量是否符合预期
- WebDAV 是否要保持开启
- 是否需要改成本地目录或 S3 / MinIO

## 7. 做一次上线前验收

至少确认下面几项：

- 浏览器可以正常登录和退出
- 文件可以上传、下载、移动到回收站并恢复
- 分享链接可以打开
- `http://127.0.0.1:3000/health` 返回正常
- 如果你启用了 WebDAV，客户端可以成功连接并读写
